    Using the right editor

    Why?

    • The right editor can make a big difference.
    • A good editor will:
      • enable code snippets
      • have bibliography intergration
      • let you compile LaTeX directly
      • save you a lot of typing

    Why?

    • An excellent text editor will:
      • work cross platform
      • be easily customisable
      • Have direct git integration
      • do half the work for you

    Which editor?

    Atom

    • https://atom.io
    • "A hackable text editor for the 21st Century"
    • Developed by Github
    • Open source, highly customisable

    Sublime Text

    • Similar to Atom, but proprietary (i.e. not open source)
    • Not free
    • Kinda free...
    • Less buggy
    • But, packages go missing in my experience

    VS Code

    • Made by Microsoft, but open source
    • Probably the best all-round text editor
    • Excellent git integration (Microsoft own Github)
    • Very stable
    • Very easy to customise
    • LaTeX integration not great: possible, but difficult.

    Emacs

    • Long history, big community
    • Relatively small installation
    • looooooots beyond LaTeX
    • You can play tetris/snake etc!
    • You can play tetris/snake etc :(
    • Not intutive to customise (lisp script)
    • Buffers instead of multiple files

    Vim

    • Terminal editor
    • Long history, large community
    • Small installation
    • You look objectively more awesome typing in a terminal window.
